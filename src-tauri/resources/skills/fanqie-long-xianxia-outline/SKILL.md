# 番茄长篇东方仙侠大纲

## 工作流程

1. 先确认目标产物：在 `选题方向`、`核心设定`、`前30章大纲`、`长线卷纲`、`后续剧情推进`、`样本节奏分析` 中选择一种；用户已经说清楚时直接执行。
2. 生成选题、节奏分析或大纲前，先完整阅读 `references/fanqie-long-xianxia-patterns.md`。
3. 分清证据与推演：本地对标素材只有每本前10章，不是完整全文；只把前10章写成“样本已验证内容”，后续卷纲必须标注为“基于开头机制的创作推演”，不得冒充原作剧情。
4. 先选定一个东方仙侠主引擎：凡骨逆袭、资源复制、宗门关系、神话改写、长生时间、游历见闻、身份反差、经营创造或规则辩经。再配置一个放大器：师徒/兄弟关系、宗门组织、仙庭背景、熟悉神话节点、市场交易或时代更迭。
5. 围绕阶段闭环搭大纲：第1章亮身份与生存钩子，第2-3章证明特殊性，第4-10章完成第一次身份重估，第11-20章接入宗门/坊市/神怪秩序，第21-30章解决首个大危机并打开长期修行目标。
6. 输出可直接开写的产物，优先写清逐章事件、修行收益、关系变化、代价与章末悬念，不写只有“修炼、历练、突破”的空壳大纲。

## 输出规则

- 除非用户另有要求，否则全部使用中文。
- 东方仙侠必须保留修行感：资源稀缺、境界门槛、师承因果、宗门规矩、时间沉淀至少占两项，不能只换皮成玄幻系统文。
- 每3-5章让主角的眼前目标可见，每8-12章完成一次小闭环。
- 制作章节大纲时使用具体章节标题。
- 每章至少包含 `主要事件`、`修行/资源变化`、`人物关系或情绪变化`、`章末钩子`。
- 金手指或特殊身份必须有边界：触发条件、消耗、风险、隐藏必要性和升级路径至少说明三项。
- 爽点不只写境界碾压，还要使用身份重估、资源翻倍、规则破解、师长认可、组织接纳、因果回收、故人变迁等仙侠型兑现。
- 慢节奏题材也要有有效变化；游历、长生、种田章节每章仍需产生新见闻、新关系、新资源、新因果或新时间对照。
- 不要照搬样本人名、法宝、宗门、章节标题、经典对白或标志性桥段，只提取节奏和结构机制。
