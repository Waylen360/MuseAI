# 番茄历史古代大纲

## 工作流程

1. 先确认目标产物：在 `选题方向`、`核心设定`、`前30章大纲`、`长线卷纲`、`样本节奏分析`、`后续剧情推进` 中选择一种；如果用户已经说清楚，就直接执行。
2. 生成选题、节奏分析或大纲前，先阅读 `references/fanqie-long-lishi-patterns.md`。分析本地样本时必须区分证据层级：前 10 章是已读正文，后续阶段只能依据 `目录.md` 标题推断。
3. 先判断子类型，再决定速度：制度知识求生、家庭科举、县政经营、商业工业、毒士朝堂、夺嫡改史、历史名场面、宗族礼法等不能套用同一章数公式。
4. 每个想法都要锚定一个清晰卖点：身份死局、著名历史节点、制度信息差、家族生存、皇权授权、经营成果、科举晋身或反常识谋略。
5. 围绕“能力验证—资源结算—社会系统扩大”搭大纲。现代知识必须兑换成户籍、读书资格、现金、人脉、师承、官职、军权或政策权限，不能只负责震惊古人。
6. 输出可直接使用的写作产物，不写空泛理论；优先给出逐章事件、冲突来源、制度约束、爽点兑现和章末悬念。

## 输出规则

- 除非用户另有要求，否则全部使用中文。
- 制作大纲时先标明子类型、历史背景、主角身份、核心能力及其限制。
- 每 3-5 章都要让阶段目标可见，但慢热家庭/科举文允许用生活改善、亲情认可和资格获得代替强打脸。
- 制作前30章大纲时使用具体章节标题；每章包含 `主要事件`、`资源/关系变化`、`爽点/钩子`、`章末悬念`。
- 真实历史人物、礼法、官制、科举、军政和经济规则必须主动验证或阻碍主角，不能只是背景贴图。
- 不要让发明、抄诗、献策成为无成本万能钥匙；每次能力兑现都要产生利益冲突、解释压力、执行成本或更高层核验。
- 不要照搬样本剧情、人名、章节标题、标志性毒计或文句，只提取结构和机制。
- 用户要求分析本地范文时，必须明确说明范文只有开头部分；不得把目录推断写成完整全文事实。
